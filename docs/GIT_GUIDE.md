# SawakouHouse Git 操作ガイド (初心者向け)

Git は、コードの「変更履歴」を記録するためのツールです。
「間違えて消してしまった！」「昨日の状態に戻したい！」という時に、過去の状態を簡単に復元できます。

---

## 1. 開発の基本サイクル (add, commit, push)

何かコードを書き換えたら、以下の3ステップで保存します。

### ① 変更したファイルを「保存リスト」に登録する
```bash
git add .
```
*   **`add`**: どのファイルを保存するか選びます。
*   **`.`**: 「すべての変更したファイル」という意味です。特定のファイルだけなら `git add backend/main.go` のように書きます。

### ② メッセージを付けて「保存」する
```bash
git commit -m "掲示板のタイトルを変更した"
```
*   **`commit`**: 変更を確定させます。
*   **`-m`**: 後で見た時に「何をしたか」が分かるように、短いメモ（メッセージ）を添えます。

### ③ インターネット（GitHubなど）に「送信」する
```bash
git push origin main
```
*   **`push`**: 自分のパソコンにある保存データを、オンライン上の共有場所に送ります。これで他の人もあなたの変更を見られるようになります。

---

## 2. 状態を確認するコマンド

「今どうなってるんだっけ？」と思ったらこれらを使います。

### 変更されたファイルを確認する
```bash
git status
```
*   赤色なら「まだ add していない」、緑色なら「add 済み（保存待ち）」の状態です。

### 変更した「中身」を詳しく見る
```bash
git diff
```
*   どの行を消して、どの行を追加したかが表示されます。

### 過去の保存履歴（ログ）を見る
```bash
git log --oneline
```
*   これまでのコミット（保存）が一覧で表示されます。

---

## 3. チーム開発でよく使うコマンド

### 他の人の変更を取り込む
```bash
git pull origin main
```
*   **`pull`**: オンライン上にある最新のコードを、自分のパソコンにダウンロードして合体させます。

### 新しい「作業場所（ブランチ）」を作る
```bash
git checkout -b feature/add-comment
```
*   **`checkout -b`**: 本番のコードを汚さずに、新しい機能を作るための自分専用の作業スペースを作ります。

---

## 4. もし失敗したら（最終手段）

### 保存前の変更をすべてキャンセルして、元に戻す
```bash
git restore .
```
*   「書き換えたけどやっぱり全部ナシにしたい！」という時に使います。**書いたコードが消えるので注意してください。**

---

## まとめ：よく使う流れ
1.  コードを書く
2.  `git status` で何が変わったか見る
3.  `git add .` で登録
4.  `git commit -m "やったこと"` で保存
5.  `git push origin main` で送信
