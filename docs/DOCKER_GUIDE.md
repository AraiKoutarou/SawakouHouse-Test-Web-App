# SawakouHouse Docker 操作ガイド (初心者向け)

このプロジェクトでは、データベース（PostgreSQL）を **Docker** という仕組みを使って動かしています。
Docker を使うことで、自分のパソコンに直接データベースをインストールしなくても、コマンド一つで全く同じ開発環境を構築できます。

---

## 1. 開発でよく使う基本コマンド

### データベースを起動する
最初に開発を始めるときや、パソコンを再起動したあとに使います。
```bash
docker-compose up -d
```
*   **`up`**: 設定ファイル (`docker-compose.yml`) に基づいて、データベースなどのサービスを起動します。
*   **`-d`**: 「バックグラウンド（裏側）」で動かすという意味です。これを使わないと、ターミナルが Docker のログで埋まって操作できなくなります。

### データベースを停止する
開発を終えるときなどに使います。
```bash
docker-compose down
```
*   **`down`**: 起動しているサービスを安全に停止し、削除します。データ自体は「ボリューム」という場所に保存されているので消えません。

---

## 2. 状態を確認するコマンド

### ちゃんと動いているか確認する
```bash
docker-compose ps
```
*   `STATUS` が `Up` になっていれば正常です。もし `Exit 0` などになっていれば、何らかの原因で止まっています。

### ログ（エラーなど）を確認する
「バックエンドがデータベースに接続できない」などの問題が起きた時に、DB側のエラーを確認できます。
```bash
docker-compose logs -f db
```
*   **`-f`**: ログをリアルタイムで流し続けます。`Ctrl + C` で終了できます。

---

## 3. トラブルが起きたら

### データを完全にリセットしてやり直す
データベースの設定を変えたのに反映されない、データが壊れた、などの場合に最終手段として使います。
**※ 保存されていた投稿データはすべて消えるので注意してください！**
```bash
docker-compose down -v
```
*   **`-v`**: 「ボリューム（保存場所）」も一緒に削除します。そのあと再び `up -d` すれば、空のきれいなデータベースが作り直されます。

---

## 4. このプロジェクトの DB 設定
`docker-compose.yml` に書かれている、接続に必要な情報です。
（バックエンドの `db/db.go` や外部ツールから接続する際に使います）

*   **DBの種類**: PostgreSQL (バージョン 16)
*   **ホスト名**: `localhost`
*   **ポート番号**: `5433` (Docker 内では 5432 ですが、外からは 5433 でアクセスするように設定しています)
*   **ユーザー名**: `user`
*   **パスワード**: `password`
*   **データベース名**: `sawakou_board`
